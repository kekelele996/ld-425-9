## 项目主要需求

### 项目概述

【全栈Web应用】家居装修全流程管理平台。面向业主、设计师和施工方，覆盖装修设计阶段、施工进度、材料清单、预算管控和3D户型预览。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿全栈）

1. **装修项目（RenovationProject）** — 字段：项目名称、户型（一室/两室/三室/四室/别墅/复式）、面积（m²）、装修风格（Modern/Chinese/Nordic/Japanese/Industrial/Mediterranean/Minimalist）、地址、业主ID、设计师ID、施工队长ID、项目状态（Designing/Quoting/InProgress/Completed/Archived）、合同金额、开工日期、预计竣工日期
   - 关联：1个项目 → 多个设计阶段、多个材料清单、多个预算项
   - **贯穿全栈**

2. **设计阶段（DesignPhase）** — 字段：所属项目ID、阶段名称（方案设计/效果图/施工图/软装方案）、设计师ID、阶段状态（NotStarted/InProgress/Revision/Approved）、版本号、设计说明、设计文件URL列表、审核意见、审核人ID
   - 业务动作：提交设计、业主审核（通过/驳回修改）、版本迭代
   - **贯穿全栈**

3. **材料清单（MaterialItem）** — 字段：所属项目ID、材料名称、品类（瓷砖/地板/油漆/灯具/卫浴/五金/板材/其他）、规格型号、品牌、数量、单位、单价、总价（自动计算）、采购状态（NotPurchased/Ordered/Delivered/Installed）、供应商、使用空间（客厅/卧室/厨房/卫生间/阳台）
   - **贯穿全栈**

4. **预算项（BudgetItem）** — 字段：所属项目ID、预算类别（Design/Material/Labor/Furniture/Appliance/Other）、预算金额、实际花费、差异金额（自动计算）、备注
   - **贯穿全栈**

5. **施工节点（ConstructionNode）** — 字段：所属项目ID、节点名称（拆改/水电/木工/瓦工/油漆/安装/软装）、计划开始/结束日期、实际开始/结束日期、状态（Pending/InProgress/Completed/Delayed）、验收状态（Pending/Passed/Failed）、验收照片URL列表、验收说明
   - **贯穿全栈**

### 核心页面（≥ 4 个）

1. **项目总览** `/dashboard` — 消费：RenovationProject + ConstructionNode + BudgetItem
   - 在建项目卡片列表、施工进度甘特图、预算执行率环形图、待验收节点提醒
   - 使用 `<StatusBadge>`, `<ProgressBar>`, `<AlertBanner>`

2. **设计管理** `/designs` — 消费：DesignPhase + RenovationProject
   - 设计阶段列表（按项目），版本历史对比，提交审核/审核操作
   - 使用 `<StatusBadge>`, `<StepIndicator>`, `<VersionTag>`

3. **材料管理** `/materials` — 消费：MaterialItem + BudgetItem
   - 材料清单表格（按空间/品类筛选），采购进度跟踪，材料费用汇总
   - 使用 `<StatusBadge>`, `<ProgressTag>`, `<EmptyState>`

4. **施工进度** `/construction` — 消费：ConstructionNode + RenovationProject
   - 施工时间线（按节点排列），验收操作（上传照片+填写说明），延期节点高亮
   - 使用 `<StatusBadge>`, `<Timeline>`, `<StepIndicator>`

5. **预算管理** `/budget` — 消费：BudgetItem + MaterialItem
   - 预算分项列表（按类别），预算vs实际柱状图，超支预警
   - 使用 `<StatusBadge>`, `<StatCard>`, `<AlertBanner>`

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / Designer / Contractor / Owner / ProjectManager
   - 触达文件层：DB role表 + RBAC中间件 + 路由守卫 + 按钮显隐（Owner只能审核不能编辑设计）+ 类型定义

2. **操作日志**
   - 记录设计提交/审核、材料采购变更、施工验收、预算调整
   - 触达文件层：DB audit_log表 + 日志中间件 + service埋点 + 前端操作时间线 + 类型定义

3. **全局异常处理**
   - 触达文件层：后端异常过滤器 + controller异常 + 前端ErrorBoundary + API拦截器 + 错误码枚举

### 共享枚举/常量（≥ 2 组）

1. **装修风格 DecorStyle**：Modern / Chinese / Nordic / Japanese / Industrial / Mediterranean / Minimalist
   - 后端：`backend/src/types/enums.ts`；前端：`frontend/src/types/enums.ts`

2. **项目阶段状态 PhaseStatus**：NotStarted / InProgress / Revision / Approved

3. **施工节点 ConstructionPhase**：Demolition / Plumbing / Carpentry / Tiling / Painting / Installation / SoftFurnishing

4. **采购状态 PurchaseStatus**：NotPurchased / Ordered / Delivered / Installed

### 共享前端组件清单（≥ 3 个）

1. **`<StatusBadge>`** — 所有页面
2. **`<ProgressBar>`** — 仪表盘、预算管理（预算执行率）
3. **`<Timeline>`** — 施工进度（节点时间线）
4. **`<StepIndicator>`** — 设计管理（多阶段流程）
5. **`<AlertBanner>`** — 仪表盘（延期/超支预警）

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useProjectPhase`** — 封装项目阶段状态流转逻辑
2. **`usePagination`** — 通用分页
3. **`formatBudget`** — 预算金额格式化和差异计算

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色校验
3. **`auditLogMiddleware`** — 操作日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`requestLoggerMiddleware`** — 请求日志

### 技术要求

- **英文短名（用于 Docker Compose）**：`home-renovation`
- **前端端口**：18805 / 80
- **后端端口**：19305 / 3000
- **前端**：React 18 + TypeScript + Vite
- **UI 库**：Ant Design 5
- **图表**：ECharts
- **后端**：NestJS + TypeScript
- **数据库**：MySQL 8.0
- **ORM**：TypeORM
- **认证**：JWT
- **状态管理**：Zustand
- **文件上传**：Multer（设计文件、验收照片）
- **构建工具**：Vite

### 部署要求

Docker Compose 一键部署。
- 顶层 `name: home-renovation`
- 不写 `version:` 字段
- MySQL 命名卷 + healthcheck
- 后端 depends_on service_healthy
- 前端 `${FRONTEND_PORT:-80}:80`，后端 `${BACKEND_PORT:-3000}:3000`

### Git 要求

`git init` + `.gitignore`。
